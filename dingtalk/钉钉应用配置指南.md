# 钉钉应用配置指南

> 如何创建钉钉机器人并获取凭据

## 步骤 1：创建企业内部应用

1. 访问 [钉钉开发者后台](https://open-dev.dingtalk.com/)
2. 登录你的钉钉账号
3. 点击「应用开发」→「企业内部开发」→「创建应用」
4. 填写应用信息：
   - 应用名称：Cursor AI 助手
   - 应用描述：通过钉钉远程控制 Cursor IDE
   - 应用图标：上传一个图标

## 步骤 2：配置机器人能力

1. 在应用详情页，点击「应用能力」→「添加应用能力」
2. 选择「机器人」
3. 配置机器人信息：
   - 机器人名称：Cursor AI
   - 机器人描述：你的 AI 编程助手
   - 消息接收模式：选择 **Stream 模式** ✅
4. 点击「发布」

## 步骤 3：配置权限

在「权限管理」页面，添加以下权限：

### 必需权限

- ✅ 企业内机器人消息接收
- ✅ 消息内容读取权限
- ✅ 文件下载权限

### 可选权限（钉钉特色功能）

- 待办管理权限（如果要集成待办）
- 日历事件管理权限（如果要集成日历）
- 智能表格权限（如果要操作表格）

## 步骤 4：配置事件订阅

1. 点击「开发配置」→「事件订阅」
2. 订阅方式：选择 **Stream 模式** ✅
3. 订阅事件：
   - 机器人接收消息：`/v1.0/im/bot/messages/get`
4. 保存配置

## 步骤 5：获取凭据

在「应用信息」页面，找到：

| 字段 | 对应 .env | 示例 |
|------|-----------|------|
| **AppKey** | `DINGTALK_APP_KEY` | `dingxxxxxx` |
| **AppSecret** | `DINGTALK_APP_SECRET` | `xxxxxxxxxxxxx` |

**复制这两个值，填入 `.env` 文件。**

## 步骤 6：发布应用

1. 在应用概览页，点击「版本管理与发布」
2. 创建版本并提交审核
3. 审核通过后发布到企业

**注意**：企业内部应用通常可以立即使用，无需审核。

## 配置 .env 文件

```bash
cd ~/work/cursor/dingtalk-cursor-claw
vim .env
```

填入配置：

```bash
# Cursor（与飞书版相同）
CURSOR_API_KEY=your_cursor_key  # 可选，CLI 会自动用 IDE 登录状态
CURSOR_MODEL=auto               # 推荐：auto

# 钉钉应用凭据
DINGTALK_APP_KEY=dingxxxxxx
DINGTALK_APP_SECRET=xxxxxxxxxxxxx

# 语音识别（可选）
VOLC_STT_APP_ID=
VOLC_STT_ACCESS_TOKEN=

# 向量记忆（可选）
VOLC_EMBEDDING_API_KEY=
VOLC_EMBEDDING_MODEL=doubao-embedding-vision-250615
```

## 常见问题

### Q: 收不到消息？

**检查清单**：
- [ ] 应用是否已发布
- [ ] 机器人是否选择了 Stream 模式
- [ ] 事件订阅是否配置了 `/v1.0/im/bot/messages/get`
- [ ] 权限是否完整
- [ ] 群聊中是否 @了机器人

### Q: access_token 获取失败？

**可能原因**：
- AppKey 或 AppSecret 填错了
- 应用未发布
- 网络问题

**解决方法**：
1. 检查 .env 中的 AppKey/AppSecret
2. 确认应用已发布
3. 查看日志：`tail -f /tmp/dingtalk-cursor.log`

### Q: Stream 连接断开？

**自动重连**：钉钉 SDK 会自动重连，无需手动干预。

**如果一直重连失败**：
1. 检查网络连接
2. 重启服务：`bash service.sh restart`
3. 查看日志排查原因
