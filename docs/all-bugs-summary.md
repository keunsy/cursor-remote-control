## 详细 Bug 列表

### 🔴 严重 Bug（9个）

1. **Scheduler 方法调用错误** ⚠️ 致命
   - 调用不存在的 `scheduler.toggle()` 和 `scheduler.trigger()`
   - 影响：任务管理命令完全失效

2. **会话管理函数未定义** ⚠️ 致命
   - 缺失 `archiveAndResetSession`, `getSessionHistory` 等函数
   - 影响：会话相关命令全部报错

3. **并发控制变量未定义** ⚠️ 致命
   - `getLockKey()`, `busySessions`, `activeAgents` 未定义
   - 影响：`/状态` 和 `/终止` 命令失效，无法追踪进程

4. **文件下载缺少错误处理** ⚠️ 高危
   - `downloadFile()` 无 try-catch
   - 影响：下载失败导致整个消息处理崩溃

5. **并发控制未生效** ⚠️ 高危
   - `busySessions` 定义了但从未使用
   - 影响：同会话并发导致状态混乱

6. **全局异常处理缺失** ⚠️ 致命
   - 无 `uncaughtException` 和 `unhandledRejection` 处理
   - 影响：未捕获异常导致服务崩溃

7. **Inbox 文件泄漏** ⚠️ 高危
   - 下载文件永久堆积，无清理机制
   - 影响：磁盘空间耗尽

8. **消息数据完整性未校验** ⚠️ 致命
   - 直接访问 `data.conversationId` 等，无空值检查
   - 影响：API 异常时服务崩溃

9. **文件下载参数未校验** ⚠️ 高危
   - 未检查 `data.content?.downloadCode` 存在
   - 影响：图片、语音、文件处理全部失效

### 🟡 中等 Bug（7个）

10. **会话历史未同步**
    - `sessionId` 未同步到 `sessionsStore`
    - 影响：会话列表为空

11. **配置热更新缺失**
    - 修改 `.env` 后不自动生效
    - 影响：用户需手动重启

12. **图片处理错误处理缺失**
    - `case 'picture'` 无 try-catch
    - 影响：图片下载失败中断处理

13. **语音处理资源泄漏**
    - `unlinkSync(audioPath)` 无错误处理
    - 影响：临时文件清理失败报错

14. **文件下载错误处理缺失**
    - `case 'file'` 无 try-catch
    - 影响：文件下载失败中断处理

15. **项目路由防御性检查缺失**
    - 未检查 `projects[intent.project]` 存在
    - 影响：依赖上游逻辑，不够健壮

16. **Token 刷新响应数据未校验**
    - 未检查 `response.data.accessToken` 存在
    - 影响：API 格式变化时 token 为 undefined

### 🟠 安全问题（1个）

17. **群聊中可发送 API Key**
    - `/密钥` 命令无群聊检查
    - 影响：Key 可能在群聊中泄露

### 🟢 轻微问题（6个）

18. **注释不准确**
    - 写的是 `cron-jobs.json` 而非 `cron-jobs-dingtalk.json`

19. **Key 显示不友好**
    - 未设置时显示 `...`

20. **热更新日志不准确**
    - Key 为空时 `slice(-8)` 显示空串

21. **并发控制缺少等待机制**
    - 检测并发后只提示，无实际等待

22. **文件名处理不健壮**
    - `data.content.fileName` 可能为 undefined

23. **Webhook 缓存条件判断冗余**
    - 前面已检查，if 判断冗余

## 根本原因分析

### 为什么会有这么多 Bug？

#### 1. **移植策略问题**（52% 的 Bug）

```
做法：复制上层代码 → 改适配 → 运行
遗漏：底层依赖函数、全局变量、错误处理

应该：系统梳理依赖 → 完整移植 → 逐项测试
```

#### 2. **测试覆盖不足**（35% 的 Bug）

```
只测试了：
✓ 正常场景（文本消息、基础命令）
✓ 启动成功（看日志无报错）

没测试：
✗ 异常场景（网络错误、数据缺失）
✗ 边界条件（空值、并发、文件不存在）
✗ 真实使用（逐个命令验证）
```

#### 3. **防御性编程意识**（13% 的 Bug）

```
问题代码：
const id = data.conversationId;
console.log(id.slice(0, 10));  // 信任 id 存在

应该：
if (!data?.conversationId) return;
const id = data.conversationId;
console.log(id.slice(0, 10));
```
