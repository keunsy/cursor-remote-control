# 钉钉远程控制技术方案

> 基于 feishu-cursor-claw 的钉钉版实现方案
> 创建时间：2026-03-11

## 可行性分析

### ✅ 技术可行性：100%

钉钉的 **Stream 模式**和飞书的 **WebSocket 模式**本质相同：
- 都是长连接（基于 WebSocket）
- 都无需公网 URL 或内网穿透
- 都支持实时消息推送
- 都支持机器人消息订阅

### 已有基础

1. **MCP 集成**：已有 4 个钉钉 MCP 服务
   - `user-dingding-doc` - 文档操作
   - `user-dingding-todo` - 待办管理
   - `user-dingding-ai-sheet` - 智能表格
   - `user-dingding-calendar` - 日历事件

2. **成熟架构**：飞书方案已完全跑通
   - Cursor CLI 调用方式
   - 会话管理机制
   - 记忆系统
   - 定时任务
   - 心跳检查

## 技术架构

### 整体流程

```
手机钉钉 ──Stream──→ dingtalk-cursor-claw ──Cursor CLI──→ Local Cursor IDE
                              │                              │
                       ┌──────┼──────┐                 --resume (session continuity)
                       │      │      │
                    Text   Image   Voice
                                    │
                         火山引擎 STT (primary)
                         Local whisper (fallback)
                                    │
                             ┌──────┴──────┐
                          Scheduler    Heartbeat
                          (cron-jobs)  (.cursor/HEARTBEAT.md)
```

### 与飞书方案的差异

| 组件 | 飞书实现 | 钉钉实现 | 工作量 |
|------|---------|---------|--------|
| **长连接** | `@larksuiteoapi/node-sdk` | 钉钉 Stream SDK | 🟡 中等 |
| **消息订阅** | `im.message.receive_v1` | `/v1.0/im/bot/messages/get` | 🟡 中等 |
| **消息发送** | 飞书 API | 钉钉 API | 🟢 简单 |
| **文件下载** | 飞书 API | 钉钉 API | 🟢 简单 |
| **语音识别** | 火山引擎 + whisper | 完全复用 | 🟢 无需改动 |
| **Cursor 调用** | agent CLI | 完全复用 | 🟢 无需改动 |
| **会话管理** | Map<key, session> | 完全复用 | 🟢 无需改动 |
| **记忆系统** | memory.ts | 完全复用 | 🟢 无需改动 |
| **定时任务** | scheduler.ts | 完全复用 | 🟢 无需改动 |
| **心跳系统** | heartbeat.ts | 完全复用 | 🟢 无需改动 |

**总工作量估算：** 70-80% 的代码可以直接复用，核心工作在钉钉 Stream 接入和消息协议适配。

## 实现步骤

### Phase 1: 核心桥接（MVP）

#### 1.1 钉钉 Stream 连接

```typescript
// dingtalk-stream.ts
import * as dingtalk from 'dingtalk-stream';

export class DingTalkStream {
  private client: dingtalk.DingTalkStream;
  
  async connect() {
    this.client = new dingtalk.DingTalkStream({
      clientId: process.env.DINGTALK_APP_KEY,
      clientSecret: process.env.DINGTALK_APP_SECRET,
      subscriptions: [
        {
          type: 'CALLBACK',
          topic: '/v1.0/im/bot/messages/get'
        }
      ]
    });
    
    // 注册消息处理器
    this.client.registerCallbackListener(
      '/v1.0/im/bot/messages/get',
      async (event) => {
        await this.handleMessage(event);
        return { code: 200, message: 'OK' };
      }
    );
    
    await this.client.connect();
  }
  
  async handleMessage(event: any) {
    const { conversationId, senderId, text, msgtype } = event;
    
    // 路由到 Cursor
    const workspace = this.getWorkspace(conversationId);
    await bridge.executeCursorTask(workspace, text, senderId);
  }
}
```

## 核心代码框架

### server.ts（主服务）

```typescript
import { DingTalkStream } from './dingtalk-stream';
import { CursorBridge } from './bridge'; // 完全复用飞书版
import { Scheduler } from './scheduler'; // 完全复用
import { HeartbeatService } from './heartbeat'; // 完全复用

class DingTalkCursorService {
  private stream: DingTalkStream;
  private bridge: CursorBridge;
  private scheduler: Scheduler;
  private heartbeat: HeartbeatService;
  
  async start() {
    // 1. 启动 Cursor 桥接（与飞书完全相同）
    this.bridge = new CursorBridge();
    
    // 2. 启动定时任务调度（与飞书完全相同）
    this.scheduler = new Scheduler(this.bridge);
    await this.scheduler.start();
    
    // 3. 启动心跳系统（与飞书完全相同）
    this.heartbeat = new HeartbeatService(this.bridge);
    await this.heartbeat.start();
    
    // 4. 启动钉钉 Stream 连接（新实现）
    this.stream = new DingTalkStream({
      onMessage: this.handleMessage.bind(this),
      onError: this.handleError.bind(this)
    });
    await this.stream.connect();
    
    console.log('钉钉长连接已启动，等待消息...');
  }
  
  async handleMessage(event: DingTalkMessageEvent) {
    const { conversationId, senderId, text, msgtype } = event;
    
    // 处理命令（复用飞书版逻辑）
    if (text.startsWith('/')) {
      return this.handleCommand(event);
    }
    
    // 项目路由（复用飞书版逻辑）
    const { workspace, message } = this.parseProjectPrefix(text);
    
    // 调用 Cursor（完全复用）
    await this.bridge.execute({
      workspace,
      message,
      sessionKey: `${conversationId}_${senderId}`,
      onProgress: (chunk) => {
        // 实时更新钉钉卡片
        this.stream.updateCard(conversationId, chunk);
      }
    });
  }
}
```

### dingtalk-stream.ts（新建）

```typescript
import * as dingtalk from 'dingtalk-stream';
import axios from 'axios';

export class DingTalkStream {
  private client: dingtalk.DingTalkStream;
  private accessToken: string;
  
  constructor(options: {
    onMessage: (event: any) => Promise<void>;
    onError: (error: Error) => void;
  }) {
    // 初始化 Stream 客户端
    this.client = new dingtalk.DingTalkStream({
      clientId: process.env.DINGTALK_APP_KEY!,
      clientSecret: process.env.DINGTALK_APP_SECRET!,
      subscriptions: [
        {
          type: 'CALLBACK',
          topic: '/v1.0/im/bot/messages/get'
        }
      ]
    });
    
    // 注册消息监听器
    this.client.registerCallbackListener(
      '/v1.0/im/bot/messages/get',
      async (event) => {
        try {
          await options.onMessage(event);
          return { code: 200, message: 'OK' };
        } catch (error) {
          options.onError(error);
          return { code: 500, message: error.message };
        }
      }
    );
  }
  
  async connect() {
    await this.refreshAccessToken();
    await this.client.connect();
  }
  
  // 获取 access_token
  async refreshAccessToken() {
    const response = await axios.get(
      'https://api.dingtalk.com/v1.0/oauth2/accessToken',
      {
        params: {
          appKey: process.env.DINGTALK_APP_KEY,
          appSecret: process.env.DINGTALK_APP_SECRET
        }
      }
    );
    this.accessToken = response.data.accessToken;
  }
  
  // 发送消息（Markdown）
  async sendMessage(conversationId: string, content: string) {
    await axios.post(
      `https://api.dingtalk.com/v1.0/robot/oToMessages/batchSend`,
      {
        robotCode: process.env.DINGTALK_ROBOT_CODE,
        msgKey: 'sampleMarkdown',
        msgParam: JSON.stringify({
          content,
          title: 'Cursor AI'
        }),
        openConversationIds: [conversationId]
      },
      {
        headers: {
          'x-acs-dingtalk-access-token': this.accessToken
        }
      }
    );
  }
  
  // 更新交互式卡片（实时进度）
  async updateCard(conversationId: string, content: any) {
    // 使用钉钉互动卡片 API
    // 类似飞书的流式更新
    await axios.post(
      'https://api.dingtalk.com/v1.0/card/instances',
      {
        outTrackId: content.messageId,
        cardData: {
          cardParamMap: {
            thinking: content.thinking,
            toolCalls: JSON.stringify(content.toolCalls),
            response: content.response,
            elapsedTime: content.elapsedTime
          }
        }
      },
      {
        headers: {
          'x-acs-dingtalk-access-token': this.accessToken
        }
      }
    );
  }
  
  // 下载文件（图片/语音/文件）
  async downloadFile(downloadCode: string): Promise<Buffer> {
    const response = await axios.get(
      `https://api.dingtalk.com/v1.0/robot/messageFiles/download`,
      {
        params: { downloadCode },
        headers: {
          'x-acs-dingtalk-access-token': this.accessToken
        },
        responseType: 'arraybuffer'
      }
    );
    return Buffer.from(response.data);
  }
}
```

## 技术风险与应对

### 风险 1: 钉钉 Stream SDK 稳定性

**评估**：钉钉官方维护，成熟度高  
**应对**：参考官方示例，测试连接稳定性

### 风险 2: 钉钉卡片渲染限制

**评估**：可能有与飞书类似的卡片大小限制  
**应对**：复用飞书版的输出控制逻辑（已在 `agent-identity.mdc` 中定义）

### 风险 3: 消息频率限制

**评估**：钉钉可能有消息发送频率限制  
**应对**：
- 合并短时间内的多次更新
- 批量更新卡片而非逐次推送
- 使用流式卡片更新而非多条消息

### 风险 4: 群聊权限管理

**评估**：钉钉群聊需要@机器人才能触发  
**应对**：
- 在规则中明确说明
- 敏感命令（如 `/密钥`）在群聊自动拦截（复用飞书版逻辑）
