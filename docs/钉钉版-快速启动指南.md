## 三步启动

### 步骤 1: 创建钉钉应用（20分钟）

参考：`~/work/cursor/dingtalk-cursor-claw/钉钉应用配置指南.md`

快速流程：
1. 访问 https://open-dev.dingtalk.com/
2. 创建企业内部应用
3. 添加机器人能力（选择 **Stream 模式**）
4. 配置权限：机器人消息接收、内容读取、文件下载
5. 事件订阅：`/v1.0/im/bot/messages/get`（Stream 模式）
6. 记录 **AppKey** 和 **AppSecret**

### 步骤 2: 配置环境变量（2分钟）

```bash
cd ~/work/cursor/dingtalk-cursor-claw
vim .env
```

修改这两行：

```bash
DINGTALK_APP_KEY=你的AppKey
DINGTALK_APP_SECRET=你的AppSecret
```

其余配置保持不变（与飞书版共用）。

### 步骤 3: 启动服务（1分钟）

```bash
# 测试运行（前台）
bun run server-minimal.ts

# 或者安装自启动
bash service.sh install
```

看到以下输出说明成功：

```
┌──────────────────────────────────────────────────┐
│  钉钉 → Cursor Agent 中继服务 v1 (MVP)           │
├──────────────────────────────────────────────────┤
│  ...
└──────────────────────────────────────────────────┘

[钉钉] access_token 已刷新
钉钉 Stream 连接已建立，等待消息...
```

## 常见问题

### Q: 钉钉收不到消息？

检查清单：
- [ ] 钉钉应用已创建并发布
- [ ] 机器人已设置为 Stream 模式
- [ ] 事件订阅已配置 `/v1.0/im/bot/messages/get`
- [ ] 权限已授予（机器人消息接收）
- [ ] .env 中的 AppKey/AppSecret 正确
- [ ] 服务已启动（`bash service.sh status`）
- [ ] 群聊中需要 @机器人

### Q: access_token 获取失败？

```bash
# 查看日志
tail -f /tmp/dingtalk-cursor.log

# 检查 .env 配置
cat ~/work/cursor/dingtalk-cursor-claw/.env | grep DINGTALK
```

确认 AppKey 和 AppSecret 正确。

### Q: 两个服务会冲突吗？

不会！它们：
- 监听不同的平台（飞书 vs 钉钉）
- 使用不同的会话 key 前缀
- 写入不同的日志文件
- 通过符号链接共享核心代码（零冲突）

唯一共享的是：
- 项目配置（只读，无冲突）
- 工作区记忆（并发安全）

### Q: 如何单独管理某个服务？

```bash
# 只管理飞书
cd ~/work/cursor/feishu-cursor-claw
bash service.sh status

# 只管理钉钉
cd ~/work/cursor/dingtalk-cursor-claw

# 或者用双服务脚本
bash ~/work/cursor/manage-both-services.sh status
```
