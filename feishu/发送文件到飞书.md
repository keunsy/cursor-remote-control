# 发送本地文件到飞书

## 🎯 功能说明

可以将本机的任何文件（如 APK、PDF、文档等）发送到飞书，方便在手机上下载使用。

## 📋 使用步骤

### 步骤 1：获取接收人 ID

首先需要知道要发送给谁。有两种方式：

#### 方式 A：发送给自己（推荐）

在飞书中给机器人发送一条消息，然后查看服务日志：

```bash
cd /Users/keunsy/work/cursor/cursor-remote-control/feishu
bash service.sh logs
```

在日志中找到类似这样的信息：
```
[消息] chat_id=oc_xxx123456 或 open_id=ou_xxx123456
```

这个就是你的 ID。

#### 方式 B：发送到群聊

如果要发送到群聊，群聊的 chat_id 格式通常是：`oc_xxx...`

### 步骤 2：发送文件

使用脚本发送文件：

```bash
cd /Users/keunsy/work/cursor/cursor-remote-control/feishu

# 发送 APK 文件（需要指定接收人ID）
bun run send-file.ts ~/app.apk ou_xxx123456

# 发送 PDF 文件
bun run send-file.ts /path/to/document.pdf ou_xxx123456

# 发送任意文件
bun run send-file.ts /path/to/file.zip oc_xxx123456
```

## 📝 完整示例

### 示例 1：发送安卓 APK 到手机

```bash
# 1. 假设你的 APK 文件在这里
ls ~/Downloads/myapp.apk

# 2. 发送给自己（替换 ou_xxx 为你的 open_id）
cd /Users/keunsy/work/cursor/cursor-remote-control/feishu
bun run send-file.ts ~/Downloads/myapp.apk ou_1234567890abcdef

# 3. 在手机飞书上收到文件，点击下载安装
```

### 示例 2：快速测试

```bash
# 发送一个小文件测试
echo "test" > /tmp/test.txt
cd /Users/keunsy/work/cursor/cursor-remote-control/feishu
bun run send-file.ts /tmp/test.txt YOUR_ID_HERE
```

## ⚙️ 高级用法：设置默认接收人

如果你经常发送文件给自己，可以编辑脚本设置默认接收人：

```bash
# 编辑 send-file.ts 文件
nano send-file.ts

# 找到第 75 行左右：
const to = receiverId || "YOUR_CHAT_ID_HERE";

# 替换成你的实际 ID：
const to = receiverId || "ou_1234567890abcdef";

# 保存后，就可以直接发送，不需要每次输入ID：
bun run send-file.ts ~/file.apk
```

## 🔧 故障排查

### 问题：找不到接收人 ID

**解决方案**：
1. 确保飞书服务正在运行：`bash service.sh status`
2. 给机器人发送一条消息：`你好`
3. 查看日志：`bash service.sh logs`
4. 在日志中搜索 `chat_id` 或 `open_id`

### 问题：文件发送失败

**可能原因**：
- 文件大于 30MB
- 接收人 ID 错误
- 飞书凭据未配置

**解决方案**：
```bash
# 检查文件大小
ls -lh /path/to/file

# 检查 .env 配置
cat .env | grep FEISHU

# 测试发送小文件
echo "test" > /tmp/test.txt
bun run send-file.ts /tmp/test.txt YOUR_ID
```

## 📱 支持的文件类型

- **安卓应用**: `.apk`
- **文档**: `.pdf`, `.doc`, `.docx`, `.xls`, `.xlsx`, `.ppt`, `.pptx`
- **图片**: `.jpg`, `.png`, `.gif`, `.webp`
- **音视频**: `.mp3`, `.mp4`, `.opus`
- **压缩包**: `.zip`, `.rar`, `.7z`
- **其他**: 任意文件类型（使用 stream 类型）

## 🎉 常见使用场景

1. **开发测试**：发送刚编译的 APK 到手机安装测试
2. **文档分享**：发送 PDF/Word 文档给自己在手机上查看
3. **应急传输**：临时发送文件给自己或同事
